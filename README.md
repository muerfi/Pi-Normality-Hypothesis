# Pi Normality Hypothesis Lab

`pi_lab` is a small scientific-computation project for generating and studying
finite prefixes of the decimal expansion of π. It provides command-line tools,
compatibility scripts, tests, and a minimal Flask interface for reproducible
experiments with digit strings.

The project is intentionally modest. It helps answer questions such as:

- How often do decimal digits or fixed-length blocks occur in a finite prefix of
  π?
- Where does a specified target string first appear in the loaded prefix?
- How can finite observations be reported without confusing them with a proof of
  normality?

It does **not** prove that π is normal in base 10. It does **not** treat found
strings as messages, codes, or evidence of hidden structure. It does **not**
claim that the digits of π are random samples. π is a deterministic mathematical
constant, and base-10 normality of π remains an open problem.

## What the project does

- Generates a local file containing the first `N` characters of π's digit string,
  using the repository convention that the file begins with the leading `3` and
  omits the decimal point.
- Loads a finite digit file from `data/pi_decimals.txt` or from a user-supplied
  path.
- Searches a loaded finite prefix for a decimal target string and reports the
  zero-based first position when the target is found.
- Counts overlapping blocks of configurable length, including ordinary
  single-digit frequencies.
- Produces a simple single-digit frequency plot for exploratory inspection.
- Keeps the reusable implementation in the `pi_lab` package while preserving
  thin compatibility wrappers for earlier script paths.

## What the project does not claim

Finite computation is useful, but it has a narrow interpretation. This
repository does not claim that:

- a finite prefix can prove or disprove normality of π;
- balanced single-digit frequencies are enough to establish normality;
- absence of a target string from the loaded prefix means it never occurs in π;
- presence of a target string has semantic significance;
- π's digits are generated by a random process.

All reported results are finite-prefix observations.

## Quick start

### 1. Create and activate an environment

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
```

### 2. Install the project

```bash
python -m pip install -e .
```

For development and tests, also install the test dependency set:

```bash
python -m pip install -r requirements-dev.txt
```

### 3. Generate a digit file

```bash
python -m pi_lab generate --digits 1000000 --overwrite
```

By default this writes `data/pi_decimals.txt`. The stored string starts with
`3`, so the first six stored characters are `314159`.

### 4. Search for a target string

```bash
python -m pi_lab search --sequence 314159
```

You can also search a specific file:

```bash
python -m pi_lab search --sequence 271828 --data data/pi_decimals.txt
```

### 5. Count block frequencies

Count single digits and generate a plot:

```bash
python -m pi_lab frequency --block-size 1
```

Count overlapping two-digit blocks without plotting:

```bash
python -m pi_lab frequency --block-size 2 --no-plot
```

### 6. Run tests

```bash
pytest
```

## Example compatibility commands

The original script paths remain available as wrappers around the package code:

```bash
python src/pi_generator.py
python src/sequence_search.py --sequence 314159
python src/frequency_analysis.py
python src/meaningful_sequences.py
python web/app.py
```

`src/meaningful_sequences.py` is retained for backward compatibility. Treat it
as a target-string search example, not as a claim that matched strings carry
interpretive content.

## Scientific background

A real number is normal in base 10 if, in the infinite limit, every finite block
of `k` decimal digits occurs with limiting frequency `10^-k`. For example, each
single digit should have limiting frequency `1/10`, each two-digit block should
have limiting frequency `1/100`, and the same condition must hold for every
block length.

This is an asymptotic property of the infinite decimal expansion. A million, a
billion, or even many trillions of checked digits can be consistent with
normality, but no finite prefix can prove the limiting statement. Single-digit
balance is especially weak: normality requires all finite block lengths, not just
individual digits.

The digits of π pass many large empirical checks, and it is widely expected that
π is normal in base 10 and in other integer bases. However, no proof is known for
base-10 normality of π.

See `docs/SCIENTIFIC_NOTES.md` for a more careful discussion and
`docs/REFERENCES.md` for background reading.

## Limitations

- The default workflow uses finite local files and therefore studies only a
  prefix of π.
- Digit generation depends on the precision and behavior of `mpmath` in the
  local Python environment.
- Block counts become sparse quickly: for block size `k`, there are `10^k`
  possible decimal blocks.
- The plotting helper is intended for quick inspection, not publication-quality
  statistical inference.
- The web interface is a lightweight local interface, not a production service.
- Search results are exact string-search results over the loaded file only.

## Reproducibility notes

For reproducible runs, record at least the following:

- repository commit;
- Python version;
- package versions, especially `mpmath`, `matplotlib`, and `flask`;
- command-line arguments, including `--digits`, `--data`, `--block-size`, and
  `--output`;
- whether `data/pi_decimals.txt` was generated locally or supplied externally;
- checksum and length of any digit file used for analysis.

A simple checksum command is:

```bash
python - <<'PY'
from pathlib import Path
import hashlib
path = Path('data/pi_decimals.txt')
data = path.read_bytes()
print(f'path={path}')
print(f'bytes={len(data)}')
print(f'sha256={hashlib.sha256(data).hexdigest()}')
PY
```
